üìã Common DNS Record Types
A (Address Record)
Maps domain to IPv4 address
Example: example.com. IN A 192.0.2.1
AAAA (IPv6 Address Record)
Maps domain to IPv6 address
Example: example.com. IN AAAA 2001:db8::1
CNAME (Canonical Name)
Alias from one domain to another
Example: www.example.com. IN CNAME example.com.
MX (Mail Exchange)
Specifies mail server for domain
Example: example.com. IN MX 10 mail.example.com.
TXT (Text Record)
Arbitrary text data (SPF, DKIM, verification)
Example: example.com. IN TXT "v=spf1 include:_spf.google.com ~all"
NS (Name Server)
Delegates subdomain to nameservers
Example: example.com. IN NS ns1.example.com.
üìù Additional Record Types
PTR (Pointer Record)
Reverse DNS lookup (IP to domain)
Example: 1.2.0.192.in-addr.arpa. IN PTR example.com.
SOA (Start of Authority)
Zone authority and configuration info
Contains: primary NS, admin email, serial, refresh, retry, expire, TTL
SRV (Service Record)
Specifies location of services
Example: _sip._tcp.example.com. IN SRV 10 60 5060 sipserver.example.com.
CAA (Certification Authority Authorization)
Specifies which CAs can issue certificates
Example: example.com. IN CAA 0 issue "letsencrypt.org"
DNSKEY / DS (DNSSEC)
Public keys for DNSSEC validation
Used for secure DNS authentication
üí° TTL (Time To Live): Specifies how long a record can be cached (in seconds)

üìä DNS Record Syntax
General Format
name TTL class type data

Example:
example.com. 3600 IN A 192.0.2.1
MX Record Format
domain TTL IN MX priority mailserver

Example:
example.com. 3600 IN MX 10 mail.example.com.
Lower priority number = higher priority
SRV Record Format
_service._proto.name TTL IN SRV priority weight port target

Example:
_sip._tcp.example.com. 3600 IN SRV 10 60 5060 sipserver.example.com.
Class: Almost always "IN" (Internet). Other classes (CH, HS) are rarely used.
üîÑ DNS Resolution Process
Recursive DNS Query Flow
User types "example.com" in browser
Browser checks its own DNS cache
OS checks system DNS cache (/etc/hosts, local cache)
Query sent to configured DNS resolver (ISP or public DNS)
Resolver checks its cache
If not cached, resolver queries root nameserver (.) for .com TLD
Root server responds with .com TLD nameservers
Resolver queries .com TLD nameserver for example.com
TLD server responds with authoritative nameservers for example.com
Resolver queries authoritative nameserver for example.com
Authoritative server responds with IP address
Resolver caches result and returns IP to client
Browser connects to IP address
Caching Levels:
1. Browser cache (short TTL)
2. OS cache (system resolver)
3. Router cache (local network)
4. ISP/Public DNS cache (configured resolver)
Each level reduces query time significantly
üéØ DNS Query Types
Recursive Query
Client asks resolver to find the answer completely
Resolver does all the work and returns final result
Most common type for end users
Iterative Query
Server returns best answer it has or referral to another server
Client must follow referrals
Used between DNS servers
Non-Recursive Query
Query for cached data only
Server returns cached answer or error
No additional lookups performed
üîê DNS Security (DNSSEC)
What is DNSSEC?
DNS Security Extensions add cryptographic signatures to DNS records
Prevents DNS spoofing and cache poisoning attacks
Validates that DNS responses are authentic
DNSSEC Record Types
DNSKEY: Public key for zone
RRSIG: Signature for record set
DS: Delegation signer (links parent to child zone)
NSEC/NSEC3: Proof of non-existence
dig +dnssec example.com
Check if DNSSEC is enabled
dig DNSKEY example.com
Query DNSSEC public keys
üí° Best Practice: Enable DNSSEC for your domains to prevent DNS hijacking

‚ö†Ô∏è Common DNS Issues & Solutions
DNS Propagation Delays
Issue: Changes not visible immediately
Cause: TTL caching at various levels
Solution: Lower TTL before changes, wait for old TTL to expire
NXDOMAIN (Domain Not Found)
Issue: Domain doesn't exist
Cause: Typo, expired domain, or misconfigured NS records
Solution: Verify domain spelling, check registration, verify NS records
SERVFAIL (Server Failure)
Issue: DNS server error
Cause: Nameserver down, DNSSEC validation failure, misconfiguration
Solution: Check nameserver status, verify DNSSEC chain, review zone file
Slow DNS Resolution
Issue: Queries taking too long
Cause: Distant nameservers, network issues, overloaded servers
Solution: Use faster DNS servers (1.1.1.1, 8.8.8.8), check network, add more nameservers
Cached Incorrect Records
Issue: Old/wrong data being returned
Cause: Stale cache entries
Solution: Flush local DNS cache, wait for TTL expiration, query authoritative server directly
