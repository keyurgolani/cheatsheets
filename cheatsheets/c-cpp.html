‚ö†Ô∏è Always initialize pointers! Uninitialized pointers can cause crashes.
üßÆ Pointer Arithmetic
int arr[] = {10, 20, 30, 40};
int *p = arr; // Points to arr[0]
printf("%d", *p); // 10
printf("%d", *(p + 1)); // 20
printf("%d", *(p + 2)); // 30
p++; // Move to next element
printf("%d", *p); // 20
Pointer arithmetic automatically scales by the size of the type. p + 1 moves by sizeof(int) bytes, not 1 byte.

üíæ Dynamic Memory (C)

if (ptr == NULL) {
// Handle allocation failure
}
// Use memory

// Free memory

malloc(size)
Allocate uninitialized memory
calloc(count, size)
Allocate zero-initialized memory

Deallocate memory
üÜï Dynamic Memory (C++)
// Single object
int *ptr = new int; // Allocate
*ptr = 42;
delete ptr; // Deallocate
// Array
int *arr = new int[10]; // Allocate array
arr[0] = 42;
delete[] arr; // Deallocate array
// With initialization
int *p = new int(42); // Initialize to 42
‚ö†Ô∏è Always use delete[] for arrays allocated with new[]. Mismatched new/delete causes undefined behavior!
‚ö†Ô∏è Common Memory Pitfalls
Memory Leak
Forgetting to free/delete allocated memory
Dangling Pointer
Using pointer after memory is freed
Double Free
Freeing the same memory twice
Buffer Overflow
Writing beyond allocated memory bounds
Null Pointer Dereference
Dereferencing a NULL/nullptr pointer
Uninitialized Memory
Reading memory before initialization
üí° Use tools like Valgrind (Linux) or AddressSanitizer to detect memory errors during development.

üéØ Smart Pointers (C++11)
#include <memory>
// unique_ptr - exclusive ownership
unique_ptr<int> ptr1 = make_unique<int>(42);
// Can't copy, only move
unique_ptr<int> ptr2 = move(ptr1);
// shared_ptr - shared ownership
shared_ptr<int> sp1 = make_shared<int>(42);
shared_ptr<int> sp2 = sp1; // Both own the object
// weak_ptr - non-owning reference

üí° Smart pointers automatically manage memory. Prefer them over raw pointers for dynamic allocation.
üîß References
// Reference - alias to existing variable
int x = 10;
int &ref = x; // ref is alias for x
ref = 20; // x is now 20
// Function with reference parameter
void increment(int &n) {
n++; // Modifies original
}
// Const reference (efficient for large objects)
void print(const string &str) {
cout << str; // Can't modify str
}
References must be initialized when declared and cannot be reassigned to refer to a different variable.

üí° Best Practices
Memory Management:
‚Ä¢ Always initialize pointers
‚Ä¢ Free/delete every malloc/new
‚Ä¢ Set pointers to NULL/nullptr after freeing
‚Ä¢ Use smart pointers in C++ when possible
‚Ä¢ Check for NULL before dereferencing
‚Ä¢ Avoid memory leaks with proper cleanup
Code Quality:
‚Ä¢ Use const correctness
‚Ä¢ Prefer references over pointers when possible
‚Ä¢ Use RAII (Resource Acquisition Is Initialization)
‚Ä¢ Avoid raw arrays, use std::vector or std::array
‚Ä¢ Enable compiler warnings (-Wall -Wextra)
‚Ä¢ Use modern C++ features (C++11 and later)
Common Mistakes to Avoid:
‚Ä¢ Buffer overflows (use bounds checking)
‚Ä¢ Dangling pointers (don't use after free)
‚Ä¢ Memory leaks (always free/delete)
‚Ä¢ Uninitialized variables
‚Ä¢ Mixing malloc/free with new/delete
‚Ä¢ Forgetting array delete[] for new[]
