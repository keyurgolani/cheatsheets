üí° Design Token Best Practices

üí° Use Semantic Names: Name tokens based on their purpose (primary, secondary) rather than their appearance (blue, red).
üí° Maintain Consistency: Use tokens consistently across all components to ensure a cohesive design system.
üí° Theme Support: Design tokens should support both light and dark themes with appropriate color adjustments.
üí° Accessibility: Ensure color contrast ratios meet WCAG standards (4.5:1 for normal text, 3:1 for large text).
üí° Responsive Spacing: Use relative units (rem, em) for spacing to support different screen sizes and user preferences.
Note: Material Design 3 (Material You) introduces dynamic color theming and updated token values. Check the official Material Design documentation for the latest specifications.
