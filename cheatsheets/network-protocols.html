üìö OSI Model (7 Layers)

Layer 7 - Application
User interface, network services (HTTP, FTP, SMTP, DNS)
Layer 6 - Presentation
Data translation, encryption, compression (SSL/TLS, JPEG)
Layer 5 - Session
Session management, authentication (NetBIOS, RPC)
Layer 4 - Transport
End-to-end connections, reliability (TCP, UDP)
Layer 3 - Network
Routing, logical addressing (IP, ICMP, ARP)
Layer 2 - Data Link
Physical addressing, error detection (Ethernet, MAC)
Layer 1 - Physical
Hardware, cables, signals (Cables, hubs, NICs)

üîó TCP/IP Model (4 Layers)

Layer 4 - Application
Combines OSI layers 5-7: HTTP, FTP, SMTP, DNS, SSH
Layer 3 - Transport
Same as OSI Layer 4: TCP, UDP
Layer 2 - Internet
Same as OSI Layer 3: IP, ICMP, ARP
Layer 1 - Network Access
Combines OSI layers 1-2: Ethernet, Wi-Fi
Mnemonic: "All People Seem To Need Data Processing" (OSI top-down)

üí° Protocol Comparison

TCP vs UDP
TCP: Reliable, ordered, connection-oriented, slower
UDP: Unreliable, unordered, connectionless, faster
Use TCP for: File transfers, web browsing, email
Use UDP for: Streaming, gaming, DNS queries
IPv4 vs IPv6
IPv4: 32-bit, 4.3B addresses, widely deployed
IPv6: 128-bit, 340 undecillion addresses, future-proof
Transition: Dual-stack, tunneling, translation
HTTP Versions
HTTP/1.1: One request per connection
HTTP/2: Multiplexing, binary protocol
HTTP/3: QUIC-based, UDP transport, faster
Email Protocols
SMTP: Sending only
POP3: Download and delete from server
IMAP: Sync across devices, keeps on server
