üîÑ Quicksort
Divide and Conquer Sorting
Best Time:
O(n log n)
Average Time:
O(n log n)
Worst Time:
O(n¬≤)
Space:
O(log n)
function quicksort(arr, low, high):
if low < high:
pivot = partition(arr, low, high)
quicksort(arr, low, pivot - 1)
quicksort(arr, pivot + 1, high)
function partition(arr, low, high):
pivot = arr[high]
i = low - 1
for j = low to high - 1:
if arr[j] < pivot:
i = i + 1
swap(arr[i], arr[j])
swap(arr[i + 1], arr[high])
return i + 1
üí° Best for: General-purpose sorting, in-place sorting. Unstable sort.
üîÄ Mergesort
Stable Divide and Conquer
Best Time:
O(n log n)
Average Time:
O(n log n)
Worst Time:
O(n log n)
Space:
O(n)
function mergesort(arr):
if length(arr) <= 1:
return arr
mid = length(arr) / 2
left = mergesort(arr[0:mid])
right = mergesort(arr[mid:end])
return merge(left, right)
function merge(left, right):
result = []
while left and right not empty:
if left[0] <= right[0]:
append left[0] to result
else:
append right[0] to result
append remaining elements
return result
üí° Best for: Stable sorting, linked lists, external sorting. Guaranteed O(n log n).
üèîÔ∏è Heapsort
Heap-Based Sorting
Best Time:
O(n log n)
Average Time:
O(n log n)
Worst Time:
O(n log n)
Space:
O(1)
function heapsort(arr):
n = length(arr)
// Build max heap
for i = n/2 - 1 down to 0:
heapify(arr, n, i)
// Extract elements from heap
for i = n - 1 down to 1:
swap(arr[0], arr[i])
heapify(arr, i, 0)
function heapify(arr, n, i):
largest = i
left = 2 * i + 1
right = 2 * i + 2
if left < n and arr[left] > arr[largest]:
largest = left
if right < n and arr[right] > arr[largest]:
largest = right
if largest != i:
swap(arr[i], arr[largest])
heapify(arr, n, largest)
üí° Best for: In-place sorting with guaranteed O(n log n). Unstable sort.
üîç Binary Search
Efficient Searching in Sorted Arrays
Time:
O(log n)
Space:
O(1) iterative, O(log n) recursive
// Iterative approach
function binarySearch(arr, target):
left = 0
right = length(arr) - 1
while left <= right:
mid = left + (right - left) / 2
if arr[mid] == target:
return mid
else if arr[mid] < target:
left = mid + 1
else:
right = mid - 1
return -1 // Not found
Requirements: Array must be sorted. Use left + (right - left) / 2 to avoid integer overflow.
üå≥ Depth-First Search (DFS)
Graph/Tree Traversal
Time:
O(V + E) for graphs, O(n) for trees
Space:
O(h) where h is height/depth
// Recursive approach
function dfs(node, visited):
if node is null or node in visited:
return
add node to visited
process(node)
for each neighbor in node.neighbors:
dfs(neighbor, visited)
// Iterative with stack
function dfsIterative(start):
stack = [start]
visited = set()
while stack not empty:
node = stack.pop()
if node not in visited:
add node to visited
process(node)
for each neighbor in node.neighbors:
stack.push(neighbor)
üí° Best for: Pathfinding, topological sorting, cycle detection, maze solving.
üìä Breadth-First Search (BFS)
Level-Order Traversal
Time:
O(V + E) for graphs, O(n) for trees
Space:
O(w) where w is max width
function bfs(start):
queue = [start]
visited = set([start])
while queue not empty:
node = queue.dequeue()
process(node)
for each neighbor in node.neighbors:
if neighbor not in visited:
add neighbor to visited
queue.enqueue(neighbor)
// With distance tracking
function bfsWithDistance(start):
queue = [(start, 0)]
visited = set([start])
while queue not empty:
node, distance = queue.dequeue()
process(node, distance)
for each neighbor in node.neighbors:
if neighbor not in visited:
add neighbor to visited
queue.enqueue((neighbor, distance + 1))
üí° Best for: Shortest path in unweighted graphs, level-order traversal, finding nearest neighbors.

üí° Algorithm Selection Guide
When to Use Each Algorithm
Quicksort: General-purpose sorting, when average-case performance matters and stability isn't required. Best for arrays.
Mergesort: When stability is required, sorting linked lists, external sorting (large datasets), guaranteed O(n log n) needed.
Heapsort: When you need guaranteed O(n log n) with O(1) space, priority queue operations, finding k largest/smallest elements.
Binary Search: Searching in sorted arrays, finding insertion points, range queries, optimization problems.
DFS: Pathfinding, topological sorting, cycle detection, maze solving, backtracking problems, tree traversal (pre/in/post-order).
BFS: Shortest path in unweighted graphs, level-order traversal, finding connected components, nearest neighbor problems.
