üìä Breadth-First Search (BFS)
Level-Order Graph Traversal
Time:
O(V + E)
Space:
O(V)
function bfs(graph, start):
queue = [start]
visited = set([start])
while queue not empty:
node = queue.dequeue()
process(node)
for each neighbor in graph[node]:
if neighbor not in visited:
add neighbor to visited
queue.enqueue(neighbor)
üí° Best for: Shortest path in unweighted graphs, level-order traversal, finding nearest neighbors.

dist = V √ó V matrix, initialized to ‚àû
for i = 0 to V - 1:

üå≤ Kruskal's Algorithm
Minimum Spanning Tree (Edge-Based)
Time:
O(E log E) or O(E log V)
Space:
O(V)
function kruskal(graph, V, E):
result = [] // MST edges
uf = UnionFind(V)
// Sort edges by weight
edges = sort(E by weight)
for each edge (u, v, weight) in edges:
if uf.find(u) != uf.find(v):
uf.union(u, v)
add edge to result
if length(result) == V - 1:
break
return result
Requires: Union-Find (Disjoint Set Union) data structure for efficient cycle detection.
üåø Prim's Algorithm
Minimum Spanning Tree (Vertex-Based)
Time:
O((V + E) log V) with min-heap
Space:
O(V)
function prim(graph, start, V):
inMST = array of size V, initialized to false
key = array of size V, initialized to ‚àû
parent = array of size V, initialized to -1
pq = min-heap with (0, start)
key[start] = 0
while pq not empty:
weight, u = pq.extractMin()
if inMST[u]:
continue
inMST[u] = true
for each (v, w) in graph[u]:
if not inMST[v] and w < key[v]:
key[v] = w
parent[v] = u
pq.insert((w, v))
return parent // MST structure
üí° Best for: Dense graphs, when starting from a specific vertex, adjacency matrix representation.
üìã Topological Sort
Linear Ordering of DAG Vertices
Time:
O(V + E)
Space:
O(V)
// DFS-based approach
function topologicalSort(graph, V):
visited = set()
stack = []
function dfs(node):
add node to visited
for each neighbor in graph[node]:
if neighbor not in visited:
dfs(neighbor)
stack.push(node)
for each vertex v in V:
if v not in visited:
dfs(v)
return reverse(stack)
// Kahn's algorithm (BFS-based)
function topologicalSortKahn(graph, V):
inDegree = array of size V, initialized to 0
// Calculate in-degrees
for each vertex u in V:
for each neighbor v in graph[u]:
inDegree[v]++
// Add vertices with 0 in-degree
queue = [v for v in V if inDegree[v] == 0]
result = []
while queue not empty:
u = queue.dequeue()
add u to result
for each neighbor v in graph[u]:
inDegree[v]--
if inDegree[v] == 0:
queue.enqueue(v)
if length(result) != V:
return "Graph has a cycle"
return result

üí° Algorithm Selection Guide
When to Use Each Algorithm
DFS: Cycle detection, topological sorting, finding connected components, maze solving, backtracking problems. Uses less memory than BFS for deep graphs.
BFS: Finding shortest path in unweighted graphs, level-order traversal, finding all nodes within k distance, testing bipartiteness.
Dijkstra: Single-source shortest path with non-negative weights, GPS navigation, network routing. Faster than Bellman-Ford when weights are non-negative.
Bellman-Ford: Graphs with negative edge weights, detecting negative cycles, distributed systems (can be parallelized), when simplicity matters over speed.
Floyd-Warshall: Finding shortest paths between all pairs of vertices, transitive closure, detecting negative cycles, small dense graphs.
Kruskal: Finding MST in sparse graphs, when edges are already sorted or easy to sort, union-find data structure is available.
Prim: Finding MST in dense graphs, when starting from a specific vertex, adjacency matrix representation, real-time MST construction.
Topological Sort: Task scheduling with dependencies, build systems, course prerequisites, resolving symbol dependencies in linkers.
‚ö†Ô∏è Important: Always verify graph properties (directed/undirected, weighted/unweighted, cyclic/acyclic) before choosing an algorithm. Some algorithms have strict requirements.
