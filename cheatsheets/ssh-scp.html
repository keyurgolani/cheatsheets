üöÄ Basic SSH Connections
ssh user@hostname
Connect to remote host
ssh user@hostname -p 2222
Connect on specific port
ssh -i ~/.ssh/key.pem user@hostname
Connect with specific private key

Execute command on remote host
ssh -v user@hostname
Verbose mode (debugging)

üîë SSH Key Management
ssh-keygen -t rsa -b 4096
Generate RSA key pair (4096 bits)
ssh-keygen -t ed25519
Generate Ed25519 key (modern, secure)
ssh-keygen -t rsa -b 4096 -C "email@example.com"
Generate key with comment
ssh-copy-id user@hostname
Copy public key to remote host
ssh-copy-id -i ~/.ssh/key.pub user@hostname
Copy specific public key
ssh-add ~/.ssh/id_rsa
Add key to SSH agent
ssh-add -l
List keys in SSH agent
ssh-add -D
Remove all keys from agent
chmod 600 ~/.ssh/id_rsa
Set correct permissions for private key
chmod 644 ~/.ssh/id_rsa.pub
Set correct permissions for public key

üîÄ Port Forwarding

Local port forwarding (access remote port 80 via local 8080)
ssh -R 8080:localhost:80 user@hostname
Remote port forwarding (expose local port 80 on remote 8080)
ssh -D 9090 user@hostname
Dynamic port forwarding (SOCKS proxy)
ssh -L 3306:db.internal:3306 user@jumphost
Access internal database through jump host
Port Forwarding Use Cases:
‚Ä¢ Access internal services through bastion host
‚Ä¢ Secure browsing through SOCKS proxy
‚Ä¢ Database connections to remote servers
üìÅ SCP File Transfer
scp file.txt user@hostname:/path/
Copy local file to remote
scp user@hostname:/path/file.txt .
Copy remote file to local
scp -r directory/ user@hostname:/path/
Copy directory recursively
scp -P 2222 file.txt user@hostname:/path/
Copy using specific port
scp -i ~/.ssh/key.pem file.txt user@hostname:/path/
Copy using specific key
scp -C file.txt user@hostname:/path/
Copy with compression
scp user1@host1:/path/file user2@host2:/path/
Copy between two remote hosts
scp -v file.txt user@hostname:/path/
Verbose mode (show progress)
üîß Advanced SCP Options
scp -p file.txt user@hostname:/path/
Preserve file attributes (timestamps, permissions)
scp -q file.txt user@hostname:/path/
Quiet mode (suppress progress)
scp -l 1000 file.txt user@hostname:/path/
Limit bandwidth (Kbit/s)
scp -o "ProxyJump=jumphost" file.txt user@target:/path/
Copy through jump host
scp *.txt user@hostname:/path/
Copy multiple files with wildcard
üí° For large transfers, consider using rsync instead:
rsync -avz -e ssh file.txt user@hostname:/path/
üîç SSH Connection Testing

Test GitHub SSH connection
ssh -vvv user@hostname
Maximum verbosity (detailed debugging)
ssh -o ConnectTimeout=10 user@hostname
Set connection timeout
ssh -o StrictHostKeyChecking=no user@hostname
Disable host key checking (use with caution)
ssh-keyscan hostname >> ~/.ssh/known_hosts

‚ö†Ô∏è Warning: Disabling host key checking can expose you to man-in-the-middle attacks. Only use in trusted environments.
üõ°Ô∏è Security Best Practices

Change passphrase for existing key
ssh-keygen -y -f ~/.ssh/id_rsa
Extract public key from private key

Security Checklist:
‚úì Use Ed25519 or RSA 4096-bit keys
‚úì Always use passphrases for private keys
‚úì Set correct permissions (600 for private, 644 for public)
‚úì Disable password authentication on servers
‚úì Use SSH agent for key management
‚úì Regularly rotate SSH keys

üîê SSH Tunneling & SOCKS
ssh -D 1080 -C -N user@hostname
Create SOCKS proxy with compression
ssh -L 5432:db.internal:5432 user@jumphost -N
PostgreSQL tunnel (no shell)
ssh -R 0.0.0.0:8080:localhost:80 user@hostname
Remote forward accessible from all interfaces
Tunnel Flags:
-N = No remote command (tunnel only)
-f = Background process
-C = Enable compression
-g = Allow remote hosts to connect
