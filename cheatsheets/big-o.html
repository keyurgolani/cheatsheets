O(log n) - Logarithmic
// Binary search
function binarySearch(arr, target) {
let left = 0, right = arr.length - 1;
while (left <= right) {
let mid = Math.floor((left + right) / 2);
if (arr[mid] === target) return mid;
if (arr[mid] < target) left = mid + 1;
else right = mid - 1;
}
return -1;
}
O(n) - Linear

// Merge sort

üí° Analysis Tips
How to Calculate:
‚Ä¢ Drop constants: O(2n) ‚Üí O(n)
‚Ä¢ Drop lower terms: O(n¬≤ + n) ‚Üí O(n¬≤)
‚Ä¢ Different inputs use different variables: O(a + b)
‚Ä¢ Nested loops multiply: O(n √ó m)
‚Ä¢ Sequential steps add: O(a + b)

Common Patterns:
‚Ä¢ Single loop = O(n)
‚Ä¢ Nested loops = O(n¬≤) or O(n √ó m)
‚Ä¢ Dividing problem in half = O(log n)
‚Ä¢ Recursion with branching = O(branches^depth)

Space Complexity:
‚Ä¢ Consider auxiliary space used by algorithm
‚Ä¢ Recursive calls use stack space
‚Ä¢ Creating new data structures increases space
